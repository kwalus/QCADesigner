QCADesigner Documentation

Installation

These instructions should guide you through the installation procedure.

Under Microsoft Windows

To run QCADesigner under Microsoft Windows download the QCADesigner self-extracting executable from the download page. QCADesigner requires a package called GTK+ to run. To this end, there are two versions of the QCADesigner Windows package on the download page. The larger one includes GTK+ and will prompt you to install it, if it doesn't find it on your computer.

If you would like to install GTK+ separately, download it from Gaim, and download the smaller of the two QCADesigner packages from the download page.

RPM Based Systems

Note: you will require the root password.

Download the RPM version of QCADesigner for your particular architecture from the download page. Login as root and go to the directory where the downloaded RPM is located. To install the RPM type:

rpm -Uhv QCADesigner-[version you downloaded]-1..rpm

That should be all you need. Now run QCADesigner with the command

QCADesigner

If for any reason the binary RPMs fail to install on your system, you can produce your own binary RPMs from the source RPM available on the download page. First, download the source RPM (QCADesigner-<version>-1.src.rpm). Then, as root, type

rpmbuild --rebuild QCADesigner-<version>-1.src.rpm

This will create binary RPMs under /usr/src/redhat/RPMS. Look for them there. Their file names will be of the form QCADesigner-<version>-1.<arch>.rpm.

From Source

Download the source from the download page. Extract the package to a known directory by typing

tar -xzvf QCADesigner-[version you downloaded]-src.tar.gz ← For tar.gz sources.

tar -xjvf QCADesigner-[version you downloaded]-src.tar.bz2 ← For tar.bz2 sources.

Switch to the new directory:

cd QCADesigner-[version you downloaded]

Prepare the program for compiling by typing

./autogen.sh

Next, you have to run the configure script to customize the compilation process for your system. The configure script provides many options. The most commonly used one is --prefix=[directory], where directory is the directory where you would like to install QCADesigner. Once installed, the QCADesigner program will be at directory/bin/QCADesigner. For example:

./configure --prefix=/home/your_account/your_install_location

Type ./configure --help for a complete list of configure options.

Compile the program with

make

Install the program with

make install

Run QCADesigner with

QCADesigner &

Voila!

Under Mac OS X

QCADesigner will run under Mac OS X, however, you will require several other packages to do this. Basically, these packages will allow you to run Unix applications on your Mac. First you will need to install Fink. When, within Fink, you choose the packages you would like installed, make sure to include the package GTK+. You will also require the X11 server. Once you have these packages you should be able to download and install QCADesigner from source. Sorry that this is not a step-by-step guide, but fink is constantly evolving, so I could end up leading you down the wrong path. You should read the installation instructions carefully on the Fink site.
